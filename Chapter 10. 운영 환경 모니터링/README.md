# 2. 모니터링 스택

### 그라파나

- 구성 가능성이 높아 에이전트를 중심으로 맞춤형 가시성을 구축하려는 팀에 유용
- 설정과 통합 : 랭그래프 애플리케이션에서 오픈텔레메트리를 초기화해 스팬과 지표 내보냄
- 핵심 기능 : 의미적 지표의 실시간 대시보드, 재시도 급증 같은 이상 징후 알림, AI 확장을 위한 강력한 커뮤니티 제공
- 트레이드오프 : 유연성과 벤더 종속성 없음

### ELK 스택(엘라스틱서치, 로그스태시, 플루언트디, 키바나)

- 강력한 검색과 분석에 중점을 둔 옵션 → 기존 엔터프라이즈 환경을 AI 워크로드로 확장하는 데 자주 활용
- 설정과 통합 : 노드를 계측해 구조화 이벤트를 로그로 남기고 엘라스틱서치의 머신러닝 작업으로 에이전트 출력의 이상 징후 탐지
- 핵심 기능 : LLM 출력에 대한 고급 전문 검색과 벡터 검색, 예측 알림을 위한 내장 ML, 클러스터링을 통한 대용량 로그 수집 확장성 제공
- 트레이드오프 : 우수한 검색, 분석 기능, 엔터프라이즈급 확장성

### 어라이즈 피닉스

- LLM 트레이싱과 평가에 초점을 맞춘 기존 환경에 에이전트 모니터링을 확장하는 디버그 지향 도구
- 설정과 통합 : 피닉스의 파이썬 SDK로 랭그래프 계측
- 핵심 기능 : 평가를 포함한 구조화 트레이싱, 머신러닝 워크플로를 위한 주피터 통합, 멀티 에이전트 조정 지표 강화
- 트레이드오프 : 평가/디버깅 특화, 프로토타이핑에 적합한 경량성

### 시그노즈

- 지표, 트레이스, 로그를 하나의 도구로 통합한 오픈텔레메트리 네이티브 플랫폼
- 설정과 통합 : 오픈텔레메트리 데이터를 직접 수집, 파이썬 자동 계측 지원
- 핵심 기능 : 에이전트 트레이스에 대한 AI 기반 인사이트, LLM 지표용 커스텀 대시보드, 클릭하우스 백엔드 기반의 경량 자체 호스팅 효율성 제공
- 트레이드오프 : 단일 앱으로 단순한 설정, 소규모 팀에 낮은 오버헤드, 강력한 오픈텔레메트리 지원과 AI 확장

### 랭퓨즈

- 파운데이션 모델과 에이전트 가시성에 특화, 에이전트용 의미 중심 트레이싱을 기존 스택에 쉽게 확장
- 설정과 통합 : SDK로 랭그래프에 통합
- 핵심 기능 : LLM 네이티브 지표, 디버깅을 위한 세션 리플레이, 데이터베이스 백엔드로의 자체 호스팅 지원
- 트레이드오프 : 에이전트/LLM 특화, 앱 수준 인사이트에 초점을 맞춰 개발 팀이 사용하기 쉬움

# 3. 프로젝트에 적합한 스택

- 그라파나 + 로키/템포
    - 조합 가능성과 시각화
    - 엔터프라이즈 운영
    - 관리해야 할 컴포넌트가 더 많음
- ELK 스택
    - 고급 검색/분석
    - 대규모 로그
    - 리소스 사용량 증가
- 피닉스
    - 트레이싱과 디버깅
    - 개발 반복
    - 프로덕션 확장 한계
- 시그노즈
    - 통합형이면서 경량
    - 스타트업/ML 팀
    - 확장성 낮음
- 랭퓨즈
    - 파운데이션 모델/에이전트 특화 평가
    - 시맨틱 모니터링
    - 인프라 커버리지 범위가 좁음

# 4. 오픈텔레메트리 계측

- 트레이스, 지표, 로그 전반에 걸친 구조화되고 상호운용 가능한 텔레메트리의 기반 제공
- 랭그래프 → 비동기 함수 호출의 그래프로 구성

# 5. 시각화와 알림

- 단순 대시보드 도구를 넘어 관측 가능성의 운영 프론트엔드로서 시그널을 스토리로, 지표를 실행으로 전환
- 에이전트의 의미론과 성공 기준에 맞춘 맞춤형 대시보드 구성 가능

# 6. 모니터링 패턴

### 섀도우 모드

- 새롭거나 실험적인 에이전트 버전을 현재 프로덕션 에이전트와 나란히 실행
- 실제 환경에서 새 에이전트의 작동 과정을 로그와 트레이스로 관찰하되 사용자 경험에는 영향 없음

### 카나리 배포

- 새 에이전트 버전을 실제 사용자 일부에게만 제공하고 나머지는 기준 버전과 상호작용
- 프로덕션 환경에서 빠르게 반복할 수 있는 운영상의 안정성 제공

### 회귀 트레이스 수집

- 실패 트레이스나 로그 스냅샷을 테스트 스위트로 자동 내보내면 지속적으로 갱신되는 회귀 코퍼스 구축 가능
- 프로덕션 실패를 학습 시그널로 전환하는 방식

### 자가 치유 에이전트

- 자체 텔레메트리를 실시간으로 읽도록 설계된 에이전트 → 문제 감지 시 폴백 메커니즘 실행 가능

# 7. 사용자 피드백

- 에이전트가 인간의 기대를 얼마나 충족하는지에 대한 직접 인사이트 제공
- 개선 루프 구동

# 8. 분포 변화

- KS 검정 : 두 데이터셋의 경험적 누적 분포함수를 비교해 동일한 분포에서 나왔는지 판단하는 비모수 검정
- 쿨백-라이블러 발산 : 한 확률분포가 다른 분포와 얼마나 다른지를 측정, 토큰 분포의 변화를 정량화해 개념 드리프트를 탐지하는 데 자주 사용
- PSI : 범주형 또는 구간화된 연속형 변수에서 과거와 현재의 비율 분포를 비교해 변화를 탐지하는 지표
