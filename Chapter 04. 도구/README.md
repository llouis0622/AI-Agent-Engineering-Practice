# 1. 랭체인 기초

### 기초

- 프롬프트를 처리하고 응답을 생성하는 파운데이션 모델을 감싼 채팅 모델 존재
- 대화 컨텍스트를 유지하기 위해 상호작용을 메시지로 구조화
- `init_chat_model()` : 모델 이름을 파라미터로 전달해 다양한 제공업체의 LLM을 동일한 채팅 모델 인터페이스로 초기화
- 메시지 유형
    - `HumanMessage` : 사용자 입력
    - `AIMessage` : 모델 응답
    - `ToolMessage` : 도구 호출 결과
- `@tool` : 도구 정의. 함수 등록 및 입력, 출력 스키마 자동 생성
- `.bind_tools()` : 모델 바인딩. 사용자 입력에 응답하는 과정에서 해당 도구를 선택해 호출 가능
- `.invoke()` : 모델과 상호작용 시 현재 대화를 나타내는 메시지 목록 전달

### 로컬 도구

- 로컬에서 실행하는 도구
- 특정 작업에 맞춘 사전 정의 규칙과 로직을 따라 생성
- 정확성, 예측 가능성, 단순성 제공
- 범위를 좁게 잡고 정확한 이름 선택
- 명확하고 구별되는 설명 작성
- 엄격한 입력/출력 스키마 정의

### API 기반 도구

- 자율 에이전트가 외부 서비스와 상호작용하도록 하여 로컬에서 수행하기 어려운 데이터 처리나 액션 실행을 가능하게 기능 확장
- 에이전트가 여러 외부 시스템과 통합하거나 실시간 데이터를 가져오거나 내부에서 처리하기엔 자원 소모가 큰 복잡한 연산을 수행해야 하는 상황에서 유리
- 실시간 데이터 접근 기능
- 신뢰성, 보안성, 우아한 실패

### 플러그인 도구

- 모듈화되어 있으며 최소한의 커스터마이징만으로 AI 에이전트 프레임워크에 통합할 수 있는 구성 요소
- 모델 실행 계층에서의 통합 가능성 → 기존 워크플로에 거의 영향을 주지 않고 AI 모델에 추가 가능

### MCP

- 모델에 독립적인 통합 표준 제공 → LLM과 외부 시스템을 연결하는 통일된 방식의 인터페이스 역할
- MCP 서버 : 표준화된 JSON-RPC 2.0 인터페이스를 통해 데이터나 서비스를 노출하는 웹 서버
- MCP 클라이언트 : MCP를 사용하는 에이전트 또는 LLM 애플리케이션

### 상태 유지 도구

- 로컬 스크립트, 외부 API, MCP에서 배포된 서비스 모두에 존재 가능
- 파운데이션 모델에 지속적인 상태를 직접 조작할 권한 부여 → 모델이 파괴적인 실수를 하거나 악성 사용자가 악용할 가능성 존재
- 최소 권한 원칙 : 모델이 꼭 필요한 도구만 사용하도록 하고 모든 작업은 명확한 경계와 감시 속에서 수행

# 2. 도구 개발 자동화

### 파운데이션 모델을 활용한 도구 개발

- LLM에 API 명세나 샘플 입력값 제공 → 초기 래퍼, 헬퍼 함수, 상위 수준의 원자적 연산 자동 생성
- 모델에 오픈API 명세서나 코드 샘플 제공 → 각 함수의 초안 자동 생성
- 명확한 검증 기준과 개발자의 감독 필수

### 실시간 코드 생성

- AI 에이전트가 작동 중 필요에 따라 코드를 작성하고 실행하는 기능
- 에이전트가 특정 작업을 수행하기 위해 새로운 도구 생성 및 기존 도구 수정 가능
- 현재 작업 분석 및 수행하기 위한 필요한 단계 결정 → 코드 스니펫 작성 및 실행
- 품질 관리, 재현성, 리소스 소비

# 3. 도구 사용 설정

- `tool-choice` : 모델의 도구 사용 방식 명시적 제어
    - `auto` : 모델이 컨텍스트에 따라 도구 호출 여부 스스로 결정
    - `any, required` : 모델이 최소 한 개의 도구를 반드시 호출
    - `none` : 모든 도구 호출 차단
- 스키마 검증 : jsonschema나 Pydantic을 사용해 누락된 필드나 잘못된 구조 탐지. 도구 호출이 누락된 경우 자동으로 실행하고 JSON이 유효하지 않다면 모델에 수정 프롬프트 전달
- 지능적 재시도 : 일시적 오류에는 지수 백오프 같은 구조화된 재시도 로직 적용 및 문제된 부분만 재생성
- 안정적인 폴백 : 재시도가 실패하면 백업 모델이나 서비스를 사용하거나 사용자에게 추가 입력을 요청하거나 캐시된 데이터를 활용하거나 안전한 기본값 반환
- 로깅 : 프롬프트, 도구 호출, 검증 오류, 재시도, 폴백 등 모든 단계를 기록해 관측 가능성을 확보하고 디버깅과 개선에 활용
