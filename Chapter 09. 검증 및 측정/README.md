# 1. 에이전틱 시스템의 측정

### 측정 : 에이전틱 시스템의 핵심

- 에이전트 시스템의 목표와 요구사항에 부합하는 명확하고 실행 가능한 지표를 식별하는 것에서 시작
- 에이전트의 작업 수행 능력과 사용자 기대 충족 여부를 평가하는 벤치마크 역할
- 정확성, 응답 시간, 견고성, 확장성, 정밀도, 재현율과 같은 정량적 지표와 사용자 만족도 같은 정성적 척도 모두 포함

### 개발 라이프사이클에 평가 통합

- 에이전트에 새로운 도구나 워크플로가 추가될 때마다 그에 상응하는 테스트 케이스와 평가 예제를 평가 세트에 추가
- 고품질 평가 세트 → 에이전트가 처리해야 할 작업을 명시하는 살아 있는 명세서 역할

### 평가 세트 생성 및 확장

- 고품질 평가 세트 : 시스템이 현실 세계에서 직면할 다양성, 모호성, 엣지 케이스 반영
- 입력 상태와 예상 결과를 모두 정의하여 에이전트 행동을 자동으로 검증할 수 있게 함

# 2. 컴포넌트 평가

### 도구 평가

- 에이전트가 환경에 작용하고 데이터를 조회하거나 변환하며 외부 시스템과 상호작용할 수 있게 하는 핵심 기능
- 가능한 사용 사례를 빠짐없이 나열하는 것에서 시작
- 각 도구마다 자동화된 테스트 모음 정의
- 같은 입력에 대해 출력이 결정적이야 한다는 점 검증 필요

### 계획 능력 평가

- 계획 모듈 : 상위 수준의 목표를 실행 가능한 단계 시퀀스로 변환
- 에이전트형 계획 : 확률적이거나 적응적이라 취약하거나 일관성 없는 작동을 피하기 위해 세심한 테스트 필요
- 도구 재현율, 도구 정밀도, 파라미터 정확도

### 메모리 평가

- 단순한 저장과 조회의 정확성을 검증하는 것뿐 아니라 메모리 저장소가 커짐에 따라 데이터 무결성, 관련성, 효율성까지 보장
- 메모리에 기록된 데이터가 정확히 저장되고 시간이 상당히 지난 뒤나 다른 작업이 개입한 뒤에도 정밀하게 다시 조회될 수 있는지 확인 필요
- 조회 로직이 오래되었거나 무관한 정보를 표면으로 끌어올리지 않도록 해야 함
- 메모리 부하가 증가하는 상황에서 조회 시간과 리소스 사용량을 벤치마킹해 성능이 급격히 떨어지는 구간이나 병목을 찾아야 함

### 학습 평가

- 에이전트가 시간이 지나면서 실제로 개선되고 단순히 과적합하거나 성능이 퇴보하거나 이미 익힌 행동을 잊어버리는 일이 없도록 보장
- 기본 학습 루프 검증, 일반화, 적응력

# 3. 총체적 평가

### 엔드투엔드 시나리오에서의 성능

- 실제 사용에 가까운 조건에서 시스템이 시작부터 끝까지 완전한 작업을 수행하는 능력을 검증하는 것
- 에이전트가 올바른 행동을 선택하는지뿐 아니라 명확하게 소통하고 사용자 의도와의 정합성을 유지하는지도 확인
- 새로운 기능, 실제 사용자 행동, 새로 나타나는 실패 양상을 반영하도록 테스트 세트를 정기적으로 확장 및 개선

### 일관성

- 에이전트의 출력이 입력과 정합성을 유지하고 장기간의 교환에서도 일관된 흐름을 이어 가며 사용자가 의도한 질문이나 작업을 안정적으로 처리하는지 확인
- 다양한 시나리오에서 에이전트의 응답이 주어진 입력과 정합성을 유지하는지 검증
- 자동화 및 수동 리뷰 모두 필요
- 액터-크리틱 접근법 → 일관성 테스트에 유용

### 응집성

- 에이전트의 출력이 상호작용 전반에 걸쳐 논리적이고 컨텍스트에 부합하며 일관되게 유지되는지 확인
- 장기 상호작용을 시뮬레이션해 에이전트가 상태에 대한 일관된 이해를 유지하는지 행동이 논리적이며 목표 지항젹 순서를 따르는지 검증

### 할루시네이션

- 에이전트가 틀렸거나 의미가 없거나 지어낸 정보를 생성할 때 발생
- 에이전트가 현실에 근거한 응답을 일관되게 생성하도록 엄격한 테스트와 완화 전략 필요
- 신뢰할 수 있는 데이터에 근거해 출력 고정 필요
- 콘텐츠 정확성, 데이터 의존성, 피드백 루프

### 예기치 않은 입력

- 예상치 못한, 잘못 구성된, 악의적인 입력에도 견고하게 작동 필요
- 에이전트가 중단되거나 유해한 출력을 만들지 않고 상황에 맞게 정중히 대처
- 무작위 퍼징만이 아니라 과거 사고나 적대적 분석에서 얻은 통찰로 엣지 케이스를 체계적으로 탐색

# 4. 배포 준비

- 운영 환경에서의 신뢰성과 신뢰도를 보장하기 위한 엄격한 준비성 점검과 품질 게이트 필요
- 관련 평가 세트에서 정량 성능 임계값을 충족하고 부하와 엣지 케이스에서 안정성을 입증하며 핵심 워크플로가 의도대로 작동함을 검증하는 항목 필요
