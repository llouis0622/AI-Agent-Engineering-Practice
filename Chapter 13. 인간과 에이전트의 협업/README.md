# 1. 역할과 자율성

### 에이전트 시스템에서 인간의 역할

- 초기 도입 단계 → 사람은 주로 실행자로서 에이전트 작업을 수동으로 시작하고 그 결과를 면밀히 감독
- 실행자 → 리뷰어 → 협력자 → 거버너로 이어지는 진전
- 협력자 → 컨텍스트 공유, 우선순위 설정, 에이전트와 실시간 결과물 생성

### 이해관계자 정렬과 도입 추진

- 이해관계자 정렬 : 엔지니어는 효율, 법무팀은 규제 준수, 최종 사용자는 사용 편의성 중시
- 설게 초기부터 이해관계자 참여
- 에이전틱 시스템 도입 → 조직 전체가 학습할 수 있는 좋은 기회

# 2. 협업 확장

### 에이전트 범위와 조직 역할

- 개인
    - 주요 사용자 : 개인
    - 접근 범위 : 이메일, 캘린더, 문서, 코드
    - 의사결정 자율성 : 낮음~보통
    - 위험 프로필 : 낮음
    - 거버넌스 요구사항 : 사용자 관리 선호 설정, 최소 감독, 설명 가능성은 선택 사항
- 팀
    - 주요 사용자 : 그룹 또는 매니저
    - 접근 범위 : 공유 드라이브, 회의, 목표
    - 의사결정 자율성 : 보통
    - 위험 프로필 : 보통
    - 거버넌스 요구사항 : 공유 메모리 경계, 동료 수준 문의 전달, 신뢰 수준 조정 필요
- 프로젝트
    - 주요 사용자 : 크로스펑셔널 그룹
    - 접근 범위 : 작업 추적, 산출물
    - 의사결정 자율성 : 보통~높음
    - 위험 프로필 : 보통~높음
    - 거버넌스 요구사항 : 크로스펑셔널 가시성, 로깅, 갈등 해결 매커니즘
- 기능
    - 주요 사용자 : 부서
    - 접근 범위 : 고객 관계 관리, 인사 정보 시스템, 재무 시스템
    - 의사결정 자율성 : 높음
    - 위험 프로필 : 높음
    - 거버넌스 요구사항 : RBAC, 감사 로그, 컴플라이언스 정합성
- 조직  전체
    - 주요 사용자 : 리더십, IT, CIO
    - 접근 범위 : 엔터프라이즈 시스템, 분석
    - 의사결정 자율성 : 높음 또는 제한적
    - 위험 프로필 : 매우 높음
    - 거버넌스 요구사항 : 다단계 승인, 거버넌스 보드 검토, 지속적인 윤리 감사와 추적성 확보

### 공유 메모리와 컨텍스트 경계

- 에이전트가 경험을 개인화하고 세션을 넘어 컨텍스트를 유지하며 시간이 지날수록 성능 개선 가능
- 메모리 적절한 범위 설정 필요
- 컨텍스트 흐름 고려

# 3. 신뢰, 거버넌스, 컴플라이언스

### 신뢰의 라이프사이클

- 일관된 성과, 투명한 행동, 명확한 경계를 통해 서서히 쌓임
- 투명성 → 신뢰 수준을 조정하는 데 핵심 역할
- 에이전트는 자신의 신뢰도, 의사결정 요인, 불확실성이 개입되었는지 여부를 선제적으로 제시 필요

### 책임성 프레임워크

- 에이전트의 행동, 의사결정, 그 결과에 대해 명확한 책임 라인이 존재하도록 보장
- 감독 정책과 문의 전달 경로 같은 구조적 조치와 함께 로깅, 추적성, 윤리 감사 같은 기술적 조치도 필요
- 출력 평가, 편향과 공정성 점검, 의사결정 경로 분석, 이해관계자 영향 평가
- 의사결정 로그, 사용자 상호작용 로그, 오류 및 실패 로그

### 대응 절차 설계와 감독

- 고위험이나 모호한 상황에서 에이전트가 권한을 넘어 행동하지 못하도록 보장하는 정책 및 인프라 레이어
- 대응 절차 경로 → 기술 시스템과 조직 역할 양쪽에 모두 인코딩
- 대응 절차 설계 → 신뢰 수준 조정에도 핵심적인 역할
- 효과적인 대응 절차 설계 → 피드백 루프 지원

### 프라이버시와 규제 컴플라이언스

- 공정성, 투명성, 책임성, 프라이버시 같은 윤리 원칙을 에이전트 설계와 배포의 구조적 요소로 넣음
- 자동화된 컴플라이언스 게이트, 코드형 정책 라이브러리, 모델 카드와 데이터시트
