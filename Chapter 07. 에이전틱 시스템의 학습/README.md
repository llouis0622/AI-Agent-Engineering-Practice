# 1. 비모수적 학습

- 관련 모델의 파라미터를 바꾸지 않고도 자동으로 성능을 변경하고 개선하는 기법

### 비모수적 예시 학습

- 에이전트가 작업을 수행하는 동안 품질에 대한 측정값을 제공하고 그 예시들을 사용해 향후 성능을 개선
- 컨텍스트 내 학습을 위한 퓨샷 예제로 사용
- 가장 관련성이 높은 예제만 동적으로 선택해 프롬프트에 포함

### 리플렉시온

- 에이전트에 언어 기반 간단한 자기 비판 습관 부여
- 각 시도가 실패한 뒤에 에이전트가 무엇이 잘못되었고 다음 시도를 어떻게 개선할지에 대해 짧은 성찰을 작성하는 방식
- 성찰 내용은 에이전트의 과거 행동 및 관찰과 함께 메모리 버퍼에 저장
- 액션 시퀀스 수행 → 시도 내역 로그에 기록 → 성찰 생성 → 메모리 업데이트 → 다음 실행에 성찰 주입

### 경험 학습(ExpeL)

- 비모수적 학습을 한 단계 더 확장
- 에이전트가 자신의 경험을 데이터베이스에 계속 모으는 것은 동일, 그 경험 전반에 걸쳐 인사이트를 집계해 미래의 정책을 개선하는 추가 단계 적용
- 경험 저장소에서 인사이트 추출 → 인사이트 목록 계속 유지 및 새로운 경험에 따라 가치 있는 인사이트는 승격, 덜 유용한 인사이트는 강등
- 작업 간 학습 과정 추가 → 리플렉시온 확장

# 2. 모수적 학습 : 파인튜닝

- 특정 작업에서의 성능을 개선하기 위해 미리 정의된 모델의 파라미터를 조정하는 방법
- 사전학습된 모델의 파라미터를 소폭 조정해 새로운 작업이나 데이터셋에 적응시키는 일반적인 접근 방식

### 대형 파운데이션 모델 파인튜닝

- 자체 포스트 트레이닝 과정 포함
- 특정 작업이나 도메인에 맞게 파라미터를 표적 조정하는 것
- 도메인 특화가 중요한 경우 : 지도 파인튜닝(SFT)이나 직접 선호 최적화(DPO)를 통해 전문성 고정 가능
- 일관된 톤과 형식이 중요한 경우 : 프롬프트 엔지니어링 없이도 올바른 구조를 안정적으로 생성하도록 가능
- 도구 및 API 호출이 매우 정확해야 하는 경우 : 함수 호출 파인튜닝을 통해 오호출을 크게 줄이고 인컨텍스트 프롬프트만 사용할 때보다 엣지 케이스 오류를 훨씬 정확하게 처리 가능
- 충분한 고품질 데이터와 예산이 있는 경우 : 데이터나 연산 자원 부족 시 리플렉시온이나 예제 검색 같은 비모수적 기법 사용
- 재학습 주기를 감당할 수 있는 경우 : 버전 관리, 재학습 일정, 호환성 검증 필요
- 지도 파인튜닝(SFT) : 선별된 쌍을 사용해 모델이 정확히 어떻게 작동해야 하는지 제공. 분류, 구조화된 출력, 지침 수행 실패 보정
- 비전 파인튜닝 : 레이블이 지정된 이미지-레이블 쌍을 주입해 모델의 멀티모달 이해 강화. 이미지 분류, 멀티모달 지침 수행 안정성
- 직접 선호 최적화(DPO) : 각 프롬프트에 대해 좋은 응답과 나쁜 응답을 함께 제공하고 어느쪽이 더 바람직한지 표시. 요약 집중도 조정, 톤/스타일 제어
- 강화 파인튜닝(RFT) : 후보 출력들을 생성하고 전문가 평가자가 점수를 매김 → 정책 경사 스타일 업데이트를 사용해 고득점 추론 과정 강화. 복잡한 추론, 도메인 특화 작업

### 소형 모델

- 더 작은 자원을 사용하는 대안으로 계산 자원이 제한되어 있거나 응답 시간이 중요한 애플리케이션에 적합
- 파라미터 수가 적고 아키텍처가 단순하지만 특정 작업에 정교하게 파인튜닝 시 매우 효과적
- 투명성, 해석 가능성 측면 이점 존재, 애자일 개발 워크플로 가능

### SFT : 지도 파인튜닝

- 모수적 접근법 중 선별된 입력/출력 예제를 통해 행동을 정밀하게 조정할 수 있게 해주는 기초적인 기법
- 함수 호출을 파인튜닝해 에이전트가 도구 호출을 올바르게 포맷하는 것뿐 아니라 호출 자체가 필요한지 여부까지 추론하도록 제작

### DPO : 직접 선호 최적화

- 선호 학습을 도입해 출력이 사람의 품질 순위 판단에 더 가깝게 정렬되도록 함
- 순위가 매겨진 응답 쌍으로부터 학습해 더 나은 출력을 덜 좋은 출력보다 선호하도록 모델을 학습시키는 파인튜닝 기법

### RLVR : 검증 가능 보상 강화 학습

- 명시적이고 측정 가능한 보상 함수에 대한 정책 최적화 도입
- 자동화 지표, 규칙 기반 검증기, 외부 스코어링 모델, 인간 평가자 등 구축할 수 있는 어떤 평가자와도 연결해 모델을 해당 보상에 직접 최적화할 수 있도록 함
