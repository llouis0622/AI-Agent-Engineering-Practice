# 1. 메모리 기본 사용법

### 컨텍스트 윈도우 관리

- 컨텍스트 윈도우 : 한 번의 호출에서 파운데이션 모델에 입력으로 전달되는 정보
- 컨텍스트 길이 : 파운데이션 모델이 한 번의 호출에서 입력으로 받아들이고 주의를 기울일 수 있는 최대 토큰 수
- 롤링 컨텍스트 윈도우 : 파운데이션 모델과의 상호작용이 진행되는 동안 전체 대화 내용을 컨텍스트 윈도우에 계속 전달. 선입선출 방식

### 전체 텍스트 검색

- 대규모 검색 및 검색 증강 시스템의 기반을 이루며 파운데이션 모델을 사용하는 에이전트에 정밀한 과거 컨텍스트를 주입하는 견고하고 성숙한 접근법 제공
- 역색인 : 모든 텍스트를 전처리하면서 토크나이징, 정규화, 불용어 제거를 수행한 뒤, 각 용어를 그 용어가 등장하는 메시지 청크나 문서 목록에 매핑
- BM25 스코어링 함수 : 각 구절을 용어 빈도, 역문서 빈도, 문서 길이 정규화에 따라 가중

# 2. 시맨틱 메모리와 벡터 스토어

### 시맨틱 검색

- 쿼리 뒤에 있는 컨텍스트와 의도를 이해하는 것을 목표로 하며 더 정확하고 의미 있는 검색 결과 제공
- 단어와 구의 의미에 초점을 맞추는 것
- 임베딩 : 대규모 텍스트 코퍼스에서의 사용 맥락을 바탕으로 단어의 의미를 포착한 벡터 표현

### 벡터 스토어로 시맨틱 메모리 구현

- 저장할 개념과 지식에 대해 시맨틱 임베딩을 생성하는 것부터 시작
- 파운데이션 모델이나 NLP 기법을 사용해 텍스트 정보를 조밀한 벡터 표현으로 인코딩
- 연속적인 벡터 공간에서 데이터 포인트의 시맨틱 속성과 관계 포착
- 벡터 데이터베이스 : 고차원 벡터 표현을 효율적으로 다루도록 설계

### RAG : 검색 증강 생성

- 검색 기반 기법과 생성 모델의 장점을 결합해 목표를 달성하는 강력한 기법
- 검색 메커니즘을 파운데이션 모델과 통합 → 더 풍부한 컨텍스트를 바탕으로 정보 생성
- 원본 문서 작은 청크로 분할 → 인코더 모델에 의해 조밀한 임베딩으로 변환 → 벡터 데이터베이스에 벡터로 저장 → 쿼리 시점에 빠른 시맨틱 조회

### 시맨틱 경험 메모리

- 각 사용자 입력이 들어올 때마다 텍스트는 임베딩 모델을 사용해 벡터 표현으로 변환
- 메모리 스토어에 저장된 과거 모든 상호작용을 대상으로 수행하는 벡터 검색의 쿼리로 사용

# 3. 그래프RAG

### 지식 그래프 활용

- neo4j-graphrag-python
- 지식 그래프 : 데이터를 그래프 형식으로 저장, 엔티티와 이들의 관계가 명시적으로 정의
- 검색 시스템 : 그래프 데이터베이스를 효율적으로 질의해 입력 쿼리나 컨텍스트와 가장 관련성 높은 서브그래프나 노드 클러스터 추출
- 생성 모델 : 그래프 형태로 검색된 관련 데이터를 기반으로 정보를 종합해 일관되고 컨텍스트가 풍부한 응답 생성

### 지식 그래프 구축

- 데이터 수집 : 소스의 다양성과 품질 확보. 에이전트의 행동에 영향을 주는 핵심 정보가 담긴 핵심 정책이나 문서 모음 수집
- 데이터 전처리 : 관려성이 없거나 중복된 정보 제거, 오류 수정, 데이터 형식 표준화
- 엔티티 인식 및 추출 : 지식 그래프에서 노드 역할을 하게 될 핵심 요소 식별
- 관계 추출 : 엔티티 사이의 관계 파악. 데이터를 파싱해 엔티티를 연결하는 술어를 추출하고 이를 그래프의 엣지로 구성하는 과정
- 온톨로지 설계 : 그래프 내의 범주와 관계 정의, 그래프의 뼈대 역할. 엔티티 유형과 이들 사이에 가능한 관계 유형을 포괄하는 스키마 정의
- 그래프 채우기 : 추출된 엔티티와 관계를 그래프에 채워 넣기. 네오4j, 오리엔트DB, 아마존 넵튠
- 통합 및 검증 : 기존 시스템과 통합 및 정확성과 유용성 검증. 다른 데이터베이스의 데이터와 연결 및 중복된 엔티티를 해소하며 그래프가 실제 지식 도메인을 정확하게 반영하는지 확인하는 작업 포함
- 유지보수와 업데이트 : 새로운 데이터 추가, 기본 정보 갱신, 새로운 유형의 엔티티나 관계에 따라 온톨로지 개선

### 동적 지식 그래프의 가능성과 위험성

- 동적 지식 그래프 : 새로운 정보를 지속적으로 업데이트되며 지식과 컨텍스트의 변화에 적응
- 들어오는 작업을 효율적으로 처리하기 위해 어떤 설계가 가장 적절한 컨텍스트를 검색해 줄 것인지를 신중하게 고려 필요
- 적응형 학습 : 주기적인 재학습이나 수동 업데이트 없이도 새로운 데이터로부터 계속해서 스스로를 갱신

### 노트 작성

- 파운데이션 모델이 질문에 바로 답하려 하지 않고 입력 컨텍스트에 대한 노트를 의도적으로 생성하도록 프롬프트
- 질문이 주어지기 전에 수행, 이후 현재 작업을 처리할 때 노트들을 원래 컨텍스트와 섞어서 함께 사용
- 셀프 노트 : 모델이 먼저 컨텍스트의 여러 부분에 대한 노트를 생성하고 그 다음 질문에 대한 노트를 만든 뒤 마지막에 최종 답변 생성
