# 1. 모달리티

### 텍스트 기반 인터페이스

- 에이전트 시스템과 상호작용하는 가장 흔하고 범용적인 방식
- 동기식 대화와 비동기식 상호작용 모두 지원
- 교환 내역이 명확하게 남음 → 투명성, 책임성, 트러블 슈팅 쉬움
- 워프 : 자연어 명령 변환, 지능형 자동완성, 컨텍스트 기반 설명 통합
- 클로드 코드 & 제미나이 CLI : 자연어 상호작용을 코드 생성, 실행, 파일 조작으로 확장
- 발견 가능성 한계 → 사용자가 에이전트가 지원하는 능력이나 효과적인 명령 표현 방식을 모르는 경우 존재
- 온보딩 메시지, 주기적 기능 안내, 대화 중 동적 제안을 통해 지원 기능 선제적 제시 필요
- 명확성, 컨텍스트 유지, 오류 관리에 대한 세심한 주의 필요
- 자연어의 모호성 → 견고한 의도 인식 필요

### 그래픽 인터페이스

- 텍스트, 버튼, 아이콘 등 여러 그래픽 요소를 결합해 사용자와 에이전트 시스템이 시각적인 상호작용을 함
- 정보를 시각적으로 제시해 인지 부하를 줄임
- 생성형 UI : 사용자 질의에 따라 인터페이스 요소, 데이터 시각화, 구조화된 출력을 동적으로 생성
- 화면의 물리적 공간이 한정적 → 중요한 정보를 노이즈 속에 파묻지 않도록 우선순위 설정 필요
- 자동화와 사용자 통제 균형 → 에이전트 자율성과 사용자 주도 행동 사이 균형 유지

### 음성 인터페이스

- 사용자가 말을 주된 소통 방식으로 에이전트 시스템과 자연스럽고 손을 쓰지 않고 상호작용하도록 제공
- 실시간 음성 처리 → 음성 인식, 의도 인식, 적절한 응답 생성 필요
- 도구 사용이 음성 에이전트 워크플로에 통합
- 사람이 음성과 문자를 처리하는 속도의 차이 이해 필요

### 비디오 기반 인터페이스

- 시각, 청각, 때로는 텍스트 요소를 하나의 일관된 경험으로 결합하는 새로운 에이전트 상호작용 방식
- 시각적 단서, 음성, 텍스트 오버레이, 애니메이션 등 여러 감각 채널을 결합해 더 풍부하고 표현력 있는 상호작용 제공
- 고품질 비디오 상호작용을 위해 상당한 연산 성능과 대역폭 필요

### 모달리티 결합을 통한 매끄러운 경험

- 여러 모달리티를 하나의 응집된 사용자 여정으로 결합
- 정보, 작업 진행 상황, 사용자 선호가 전환 과정에서 결코 유실되지 않도록 세심한 상태 관리와 컨텍스트 지속성 필요

### 자율성 슬라이더

- 완전 수동 제어에서 부분 자동화, 완전 자율 운영까지 에이전트의 자율성을 사용자가 부드럽게 조정 필요
- 사용자가 특정 시점에 유지할 통제와 위임할 범위 선택 가능
- 수동 : 개발자가 에이전트 도움 없이 모든 코드 직접 작성. IDE는 문법 하이라이트와 린팅만 제공하는 편집기 역할을 하며 AI 기반 제안은 없음
- 요청(보조) : 에이전트가 코드 자동완성, 리팩터링, 문서 스니펫을 선제적으로 제안하지만 적용 전에는 개발자가 각 제안을 검토해 수락. 속도를 높이면서 개발자가 완전한 통제권 유지
- 완전 자율 : 에이전트가 표준 리팩터링 적용, 린터 오류 수정, 프로젝트 규약에 따른 보일러플레이트 파일 생성 등 특정 작업을 개별 승인 없이 자율적으로 수행. 개발자는 변경 사항을 알림으로 받지만 각 작업 승인 불필요
- 명확한 자율성의 단계 노출: 수동에서 보조, 자율까지 가능한 에이전트 독립성 수준을 사용자가 이해
- 매끄러운 전환 가능 : 사용자 신뢰, 컨텍스트, 업무량이 변할 때 자율성 수준을 손쉽게 변경 필요
- 각 수준에서 예측 가능하고 투명한 기능 제공 : 작동 과정 잘 정의
- 각 수준의 위험과 이점 소통 : 자율성을 높일 때 얻는 이점과 감수하는 위험 사용자 인지 필요
- 사용자 신뢰와 역량에 따라 자율성 조절 : 신뢰가 쌓일수록 자율성 수준 높이도록 제안 가능

# 2. 동기식 대 비동기식 에이전트 경험

### 동기식 경험을 위한 디자인 원칙

- 즉시성, 명확성, 반응성 중요
- 응답에서 명료함과 간결함을 우선시
- 오류 처리 중요

### 비동기식 경험을 위한 디자인 원칙

- 즉시성보다 유연성, 지속성, 장기적 명료성 우선
- 작업 상태와 결과를 명확히 전달 필요
- 컨텍스트 관리 필요
- 사용자 기대를 효과적으로 관리 필요

### 능동적 개입과 침해적 행동 사이의 균형

- 에이전트가 언제, 어떻게 능동적으로 사용자를 참여시켜야 하는지 결정 어려움
- 컨텍스트 인식과 사용자 통제에 능동성의 균형이 맞춰짐

# 3. 컨텍스트 유지와 연속성

### 상호작용 간 상태 유지

- 상태 관리 → 에이전트 시스템의 컨테스트 연속성을 떠받치는 중요한 역할
- 시스템이 사용자나 세션을 어떻게 식별, 추적하느냐에 따라 효과적인 상태 관리 가능
- 명확한 세션 경계, 데이터 검증, 폴백 메커니증 요구

### 개인화와 적응성

- 선호 보존 : 알림 선호, 자주 선택하는 옵션 같은 사용자 설정 기억
- 행동 적응 : 관찰된 사용자 패턴에 기반해 응답 스타일이나 상호작용 흐름 조정
- 능동적 지원 : 과거 행동을 바탕으로 사용자 필요를 예측하고 제안 제공

# 4. 에이전트 능력 커뮤니케이션

### 신뢰도와 불확실성 커뮤니케이션

- 결정적 규칙이 아닌 통계 모델에 기반한 확률적 환경에서 작동 → 신뢰도 불확실성
- 명시적 진술, 시각적 신호, 행동 조정
- 사용자 기대와 상호작용의 중요도에 맞게 응답을 구성하는 일

### 사용자 지침과 입력 요청

- 모호하거나 애매하고 상충되는 사용자 입력 완벽히 이해 불가
- 모호성에 직면했을 때 초점을 맞춘 유용한 질문을 하도록 설계
- 질문은 명확하고 공손하며 컨텍스트 인식적이어야 하며 로봇 같거나 반복적인 표현 피해야 함
- 왜 명확화가 필요한지 투명하게 설명 필요
- 한 번에 너무 많은 질문 피하기

### 우아한 실패

- 문제를 투명하게 인정하고 유용한 설명을 제공하며 실행 가능한 다음 단계 제시 필요
- 실패 지점을 예측해 사전에 폴백 메커니즘 준비 필요
- 다단계 작업 시 실패 시 진행 상태 보존 중요
- 명확한 해결 경로 제공
- 실패 지점 기록 및 반복 이슈 분석으로 개발 프로세스 반영

# 5. 상호작용 설계에서의 신뢰

- 투명성, 예측 가능성 → 에이전트와 사용자 사이의 신뢰 구축 및 유지 가능
- 투명성 → 에이전트의 능력과 제약을 명확히 전달하는 데서 시작
- 예측 가능성 → 다양한 상황에서의 일관된 행동을 통해 투명성 보완
- 신뢰성 → 출력의 일관성 필요, 시스템 복원력
