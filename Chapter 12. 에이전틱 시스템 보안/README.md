# 1. 에이전틱 시스템만의 위험

- 목표 불일치, 확률적 추론, 동적 적응, 제한된 가시성
- 자동화 편향, 알림 피로, 역량 저하, 인센티브 불일치
- Gandalf by Lakera : 단계적으로 진화하는 AI 방어를 우회해 비밀을 추출하는 프롬프트를 설계하며 입출력 필터링과 다계층 보호 개념을 배우는 교육용 게임
- Red by Giskard : 편향, 유해성 등을 노리는 짧고 창의적인 프롬프트로 파운데이션 모델을 공략하는 난이도 상승형 인터랙티브 게임
- Prompt Airlines CTF by Wiz.io : 프롬프트 인젝션으로 항공사 고객지원 챗봇을 탈옥해 무료티켓 등 숨겨진 정보를 빼내는 CTF 방식 챌린지

# 2. 새로운 공격 수단

- 프롬프트 인젝션 : 의도된 지시를 덮어쓰거나 의도치 않은 행동을 일으키도록 에이전트를 조작하는 악성 입력
- 간접 프롬프트 인젝션 : 에이전트가 처리하는 외부 데이터에 숨긴 악성 지시가 출력물을 훼손
- 민감 정보 노출 : 필터링 미흡 등으로 에이전트 출력에 기밀이 의도치 않게 포함
- 가드레일 우회 : 파운데이션 모델의 안전 필터. 제약을 우회해 금지된 행동을 유도
- 소셜 엔지니어링 : 인간-에이전트 상호작용을 악용해 정보 유출, 행동 유도
- 회피 공격 : 입력을 변형해 필터, 분류기 등 탐지를 회피
- JSON 기반 프롬프트 인젝션 : 구조화된 JSON 형식을 로그, 설정, 내부 메시지처럼 가장해, 모델이 권위 있는 입력으로 해석하는 점을 악용
- 에이전트 스웜 악용 : 에이전트 간 조정 취약점을 이용해 위협을 증폭해 창발적 악의적 행동이나 시스템적 침해 유발

# 3. 파운데이션 모델 보안

### 방어 기법

- 악의적 악용을 차단하고 의도치 않은 오작동을 줄이며 다양한 컨텍스트에서 모델이 신뢰성 있게 작동하도록 보장하는 것
- 입력 정제와 검증, 프롬프트 인젝션 방지, 출력 필터링과 검증, 접근 제어와 요청 제한

### 레드팀

- 전문가가 적대적 공격을 시뮬레이션해 에이전트 시스템과 그 기반이 되는 파운데이션 모델의 취약점, 약점, 실패 모드를 식별하는 선제 보안 실천
- 현실 세계의 공격 전략을 모사한 적대적 시나리오를 설계 및 실행하는 일
- 포괄적 커버리지를 위해 자동화 레드팀 도구가 인간 테스터와 함께 점점 더 많이 사용
- DeepTeam, Garak, PyRIT

### MAESTRO 기반 위협 모델링

- AI 라이프사이클 전반에서 취약점을 체계적으로 식별하고, 위험을 평가하며, 완화책을 구현할 수 있도록 계층형 참조 아키텍처 제공
- 위협, 위험, 결과를 모듈 방식으로 매핑해 관심사의 분리를 보장하면서도 레이어 간 의존성을 부각해 선제 보안을 촉진

# 4. 에이전틱 시스템의 데이터 보호

### 데이터 프라이버시와 암호화

- 무단 접근, 데이터 유출, 의도치 않은 노출에 대한 1차 방어선
- 휴지 상태 : 암호화를 통해 에이전트 시스템에 저장된 민감 정보가 무단 당사자에게는 해독 불가능하도록 함
- 전송 상태 : 엔드투엔드 암호화가 에이전트, 외부 API, 스토리지 시스템 사이을 이동하는 데이터 보호
- 안전한 데이터 보관, 삭제 정책

### 데이터 출처와 무결성

- 에이전트가 의존하는 정보가 정확하고 신뢰할 수 있으며 변조되지 않았음을 보장하는 데 필수
- 데이터 라이프사이클 전반에 걸쳐 데이터가 변경되거나 위조되지 않았음을 보장하는 데 초점
- 변경 불가능 저장소나 추가 전용 로그를 사용해 출처와 무결성을 함께 강화

### 민감 데이터 처리

- 보안 데이터 처리의 기반에는 데이터 최소화 원칙 존재
- 역할 기반 접근 제어와 속성 기반 접근 제어 시스템을 구현 중요
- 암호화 프로토콜은 데이터 라이프사이클 전반에 걸쳐 적용
- 안전한 로깅과 감사, 데이터 보존 및 삭제 정책 처리

# 5. 에이전트 보안

### 보호 장치

- 에이전트의 자율성, 상호작용, 의사결정 과정과 관련된 위험을 최소화하도록 설계된 선제적 통제와 보호 수단
- 에이전트 행동 제약, 환경 격리, 에이전트 상호작용을 위한 게이트키퍼 역할을 하는 입력과 출력 검증 파이프라인
- 감사 추적 및 로깅 메커니즘, 폴백과 페일세이프 메커니즘

### 외부 위협으로부터의 보호

- 기술적 통제, 실시간 모니터링, 선제적 완화 기법을 결합한 계층화된 방어 전략 필요
- 외부에 노출된 컴포넌트와 민감한 내부 리소스를 분리하는 보안 네트워크 아키텍처
- 네트워크 보안, 인증과 인가 메커니즘
- 프롬프트 인젝션, 데이터 포이즈닝, 적대적 공격 방어 필요
- 실시간 이상 탐지 시스템, 엔드포인트 하드닝, 선제적 보안 테스트와 감사

### 내부 실패로부터의 보호

- 에이전트의 지시나 운영 목표 안에서 목표와 제약이 정렬되지 않는 경우
- 에러 처리와 예외 관리는 내부 실패에 대한 중요한 보호 장치
- 모니터링과 텔레메트리 시스템은 내부 실패를 위한 조기 경보 메커니즘 역할
- 오류율, 응답 지연시간, 리소스 활용률, 출력의 이상 점수, 상태 일관성 검사
- 결함 주입, 게임 데이와 실험, AI 특화 적응, 피해 범위 통제
